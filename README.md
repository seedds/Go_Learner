# GoLearner

An iPhone Go (囲碁 / 바둑) app powered by the **KataGo** neural network running
on the **Apple Neural Engine** via Core ML.

Play against a strong AI, watch it analyze the position live (win rate, score
lead, territory ownership, candidate moves), and step through your game — all
on-device, offline.

> Status: **MVP in progress.** All Swift + C++ + bridge code is written and the
> C++ engine subset is verified compiling & running on the host. The Xcode
> project generation and on-simulator build are the next step (see
> [Build](#build)).

---

## What it is

GoLearner is a lean, single-target iPhone app built fresh (not a fork). It
reuses the proven parts of the KataGo engine where correctness matters and
keeps everything else small and modern SwiftUI.

- **Real KataGo strength.** Moves and analysis come from the official 18-block
  `b18c384nbt` network (model version 14), the same net used by desktop KataGo.
- **Neural Engine inference.** The network is a bundled Core ML model, run on
  the ANE (`.cpuAndNeuralEngine`) for speed and power efficiency.
- **Correct Go rules & NN features.** Board rules, legality, ko/superko,
  scoring, and the 22 spatial + 19 global input features are generated by a
  vendored slice of KataGo's own C++ (`board.cpp`, `boardhistory.cpp`,
  `rules.cpp`, `nninputs.cpp`) — no reimplementation guesswork.
- **Offline & private.** The model is bundled in the app; nothing leaves the
  device.

### MVP scope

- 19×19 board, komi 7.5, area (Tromp-Taylor-ish) rules.
- Human vs AI, AI vs AI, or Human vs Human (tap a player capsule to toggle).
- One neural-net evaluation per move (raw-policy play) — strong, fast, honest.
- Live analysis overlay: win rate, score lead, ownership shading, top candidate
  moves with probabilities.
- Pass, undo, new game.

### Deliberately **not** in the MVP (easy follow-ups)

- MCTS search (the engine seam is built to accept it later).
- Multiple board sizes / rule editing.
- SGF import/export, game library, iCloud sync.
- Learning features (tsumego, guided "why this move") — the namesake direction.

## How it works (30-second version)

```
 SwiftUI (ContentView / BoardView / AnalysisOverlay)
        │  reads
        ▼
 GameState  ── owns ──►  GoBridge (ObjC++)  ──►  KataGo C++ subset
   (@MainActor,             │  fills 22×19×19 spatial + 19 global features
    @Observable)            │  + move legality / board state
        │ passes plain arrays
        ▼
 GoEngine (actor)  ──►  NNModel  ──►  Core ML  ──►  Apple Neural Engine
        ▲                    │  decodes policy / value / score / ownership
        └──── NNResult ──────┘  (math mirrors KataGo's nneval.cpp)
```

- The **C++ bridge is single-threaded** and lives on the main actor. Feature
  filling is microsecond-cheap.
- The **Core ML evaluation runs off the main thread** inside `GoEngine` (an
  `actor`), taking plain `[Float]` value types so the C++ object never crosses
  threads.
- Results are decoded with the *exact* post-processing from KataGo's
  `nneval.cpp` / `coremlbackend.cpp` so the numbers match the reference engine.

See **[ARCHITECTURE.md](ARCHITECTURE.md)** for the full map.

## Build

Requirements: macOS with **Xcode 26+** (iOS 26 SDK), Apple silicon recommended.

The Core ML model is already in `Resources/`. The `.xcodeproj` is generated
from `project.yml` with [XcodeGen](https://github.com/yonyz/XcodeGen)
(`brew install xcodegen`).

```bash
cd GoLearner
xcodegen generate          # produces GoLearner.xcodeproj
open GoLearner.xcodeproj   # or build from the command line:

xcodebuild build \
  -project GoLearner.xcodeproj \
  -scheme GoLearner \
  -destination 'platform=iOS Simulator,name=iPhone 17'
```

> Set your development team in *Signing & Capabilities* to run on a device.
